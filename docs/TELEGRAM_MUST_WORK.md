# Обязательный функционал в Telegram

Список сценариев, которые должны стабильно работать в боте.

| № | Сценарий | Статус | Где реализовано |
|---|----------|--------|------------------|
| 1 | **/start** | ✅ | `handlers/start.py`: приветствие, bootstrap админа, экран по статусу/роли |
| 2 | **Главное меню кнопками** | ✅ | `keyboards/root_menu.py`, `navigation.py`: Регистрация, AI-куратор, Админ-панель (для админа), Помощь |
| 3 | **Регистрация** | ✅ | `handlers/verification.py`: FSM роль → имя → фамилия → ТТ → телефон → подтверждение → заявка в БД |
| 4 | **Pending screen** | ✅ | `menu_renderer.py`: текст «Заявка на рассмотрении» + кнопка «Обновить статус» |
| 5 | **Уведомление админу о новой заявке** | ✅ | `proactive_layer.py`: по событию `verification.pending` отправка сообщения всем из `ADMIN_IDS` |
| 6 | **Экран верификации админа** | ✅ | `admin/admin_handlers.py`: раздел «Верификация» → список заявок (pending) → кнопки Одобрить/Отклонить |
| 7 | **Кнопка «AI куратор»** | ✅ | `keyboards/root_menu.py` (ROOT_AI_CURATOR), `handlers/ai_chat.py`: вход в режим чата с AI |
| 8 | **Чат с AI по кейсам** | ✅ | `handlers/ai_chat.py`: быстрые кейсы + свободный текст, answer_user, кнопка «Назад в меню» |
| 9 | **Админ-меню с переходами** | ✅ | `keyboards/admin_main.py`, `admin/admin_handlers.py`: Верификация, AI-куратор, Анализ CSV, Мониторинг, ТМЦ, Рассылка; «Назад» |
| 10 | **Мониторинг** | ✅ | `admin_handlers.py`: раздел «Мониторинг» — краткий статус (бот, AI вкл/выкл) |
| 11 | **Рассылка** | ✅ | Раздел «Рассылка» открывается; текст-заглушка «в будущем» (минимальный must-work) |
| 12 | **Загрузка файла на анализ** | ✅ | `admin/ingest.py`: Админ → Анализ CSV → Импорт CSV → приём документа .csv → IngestService, очередь парсинга |

## Зависимости

- **ADMIN_IDS** в `.env` — список Telegram ID администраторов (через запятую); иначе уведомления о заявках и доступ в админку не работают.
- **AI**: при `AI_ENABLED=false` кнопка «AI куратор» и чат отвечают заглушкой «AI не инициализирован».
- **БД**: миграции до 016 (минимальный production core); для верификации — таблицы `users`, `verification_applications`.

## Точки входа

- Пользователь: `/start` → экран по статусу (гость / pending / rejected / approved+роль).
- Админ (из главного меню или после /start): «Админ-панель» → разделы по кнопкам.
- Регистрация: главное меню → «Регистрация» или экран гостя → «Регистрация».
