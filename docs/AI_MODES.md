# Разделение режимов AI

В системе не один «умный ответчик», а **три явных режима** с разной зоной ответственности. Все они входят через единый фасад `AIFacade`, но вызывающие кода должны использовать правильный метод под задачу.

## 1) Courier assistant (ассистент курьера)

**Назначение:** ответы по кейсам доставки — регламент, must_match, FAQ, ML-кейсы, при необходимости LLM.

**Точки входа в коде:**
- `AIFacade.answer_user(user_id, text)` — ответ на вопрос курьера (каскад решений, см. [AI_CURATOR.md](AI_CURATOR.md)).
- `AIFacade.proactive_hint(risk_input)` — проактивная рекомендация по риску доставки (risk_engine + recommendation_engine).

**Где используется:** бот (AI-куратор в меню курьера), команда `/risk`, API автоматизаций при запросе от пользователя.

**Режим в коде:** `AIMode.COURIER_ASSISTANT` ([ai_modes.py](../src/core/services/ai/ai_modes.py)).

---

## 2) Admin copilot (копилот для админа)

**Назначение:** помощь администратору — FAQ по админке, подсказки по рассылкам, анализ, служебные вопросы.

**Точки входа в коде:**
- `AIFacade.answer_admin(admin_id, text)` — ответ в админ-чате на вопрос админа.

**Где используется:** админ-чат (когда админ пишет AI-помощнику). Сейчас пайплайн общий с курьером; в перспективе — отдельный контекст/политика (админский FAQ, темы рассылок, разбор отчётов).

**Режим в коде:** `AIMode.ADMIN_COPILOT`.

---

## 3) Analytics assistant (аналитический ассистент)

**Назначение:** анализ данных — CSV, xlsx, pdf, таблицы; формирование текстовых отчётов по метрикам.

**Точки входа в коде:**
- `AIFacade.analyze_csv(tt_metrics=..., global_summary=..., raw_sample_notes=...)` — аналитический отчёт по метрикам успешных доставок (сейчас только CSV-метрики; в перспективе — загрузка xlsx/pdf и универсальный разбор таблиц).

**Где используется:** админка (отчёты по доставкам), дашборды, выгрузки. Метрики считаются в коде/ETL; LLM только объясняет паттерны и формирует выводы в заданном формате.

**Режим в коде:** `AIMode.ANALYTICS_ASSISTANT`. Реализация: `AnalyticsAssistant` в `src/core/services/ai/analytics_assistant.py`, промпты в `data/ai/prompts/last_mile_analyst_*.md`.

---

## Сводка

| Режим                 | Методы фасада        | Контур логики                          |
|-----------------------|----------------------|----------------------------------------|
| Courier assistant     | `answer_user`, `proactive_hint` | AICourierService, каскад решений       |
| Admin copilot         | `answer_admin`       | AICourierService (пока тот же пайплайн)|
| Analytics assistant   | `analyze_csv`        | AnalyticsAssistant                     |

Все вызовы LLM из бота, API и админки — только через `AIFacade`. Прямые вызовы `AICourierService`, `ProviderRouter`, `AnalyticsAssistant` из handlers запрещены.
