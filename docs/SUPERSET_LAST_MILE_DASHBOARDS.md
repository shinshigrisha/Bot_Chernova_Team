# Дашборды Superset: аналитика последней мили

Документ описывает, как создать дашборды в Apache Superset для визуализации данных витрин последней мили (raw_orders, mart_orders_enriched, раскатка по сервисам, качество ТТ, нарушения курьеров).

---

## 1. Подключение к БД

В Superset: **Data → Databases → + Database**. Выберите **PostgreSQL** (или совместимый источник), укажите URL подключения к той же БД, где применены миграции с витринами (таблица `raw_orders`, представления `mart_*`).

---

## 2. Датасеты (Datasets)

Создайте датасеты на основе представлений и таблицы:

| Датасет в Superset | Источник (таблица/view) | Назначение |
|--------------------|-------------------------|------------|
| **Last mile — заказы (enriched)** | `mart_orders_enriched` | Нормализованные заказы со статусом «Успешно завершен» и расчётными метриками |
| **Раскатка по сервисам (куст)** | `mart_service_rollout_cluster` | Доля заказов по сервисам в разрезе по кусту |
| **Раскатка по ТТ** | `mart_service_rollout_tt` | Раскатка сервисов по ТТ |
| **Качество ТТ** | `mart_tt_quality` | Метрики качества по ТТ (просрок, сборка, доставка) |
| **Нарушения курьеров** | `mart_courier_violations` | Агрегаты по курьерам: просроки, «чистые» нарушения |

При создании датасета укажите схему (обычно `public`) и имя таблицы/представления.

---

## 3. Рекомендуемые чарты и дашборды

### Дашборд: Раскатка сервисов

- **Чарт 1:** таблица или bar chart по датасету `mart_service_rollout_cluster`: оси `service_name`, `orders_cnt`, `rollout_pct`.
- **Чарт 2:** таблица по `mart_service_rollout_tt` с фильтрами по `tt_name` и `service_name`, колонки `orders_cnt`, `rollout_pct`.

### Дашборд: Качество ТТ

- **Чарт 1:** таблица по `mart_tt_quality`: все колонки (success_orders, couriers_cnt, late_flag_pct, late_15_plus_pct, avg_delivery_wait_minutes и т.д.).
- **Чарт 2:** bar chart — ТТ по оси X, метрики (например `late_flag_pct`, `assembly_over_norm_pct`) по оси Y.
- Фильтры: при необходимости по периоду (если в витрину добавить срез по дате).

### Дашборд: Курьеры и нарушения

- **Чарт 1:** таблица по `mart_courier_violations` с сортировкой по `clean_courier_violations` или `late_pct`.
- **Чарт 2:** фильтр по `tt_name` и `ziz_name` для детализации по ЗиЗ.

---

## 4. Проверка данных

После загрузки данных в `raw_orders` (и при наличии строк со статусом «Успешно завершен»):

1. В Superset выполните запрос к `mart_orders_enriched` — должны появиться строки.
2. Проверьте датасеты `mart_service_rollout_cluster`, `mart_tt_quality`, `mart_courier_violations` — чарты должны отображать данные в соответствии с агрегатами.
