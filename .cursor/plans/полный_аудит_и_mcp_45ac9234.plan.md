# План полного аудита и починки MCP Context

## Цель

Сначала детально сверить проект с требованиями документации и планов MVP, затем устранить проблемы с `MCP Context7`, после чего предоставить итоговый gap-анализ реализации по каждому модулю.

## Что уже выявлено на этапе исследования

- Документация в `docs` присутствует и описывает MVP/архитектуру/деплой/безопасность/runbook.
- В планах `.cursor/plans` (три версии) зафиксированы одинаковые этапы MVP (0–7).
- По коду реализован базовый каркас (бот, БД, очереди, FSM), но есть недореализованные зоны (реальный ingest parse, заглушки admin-пунктов).
- Конфигурация MCP потенциально нестабильна из-за хранения ключа в файле и возможного рассинхрона ожиданий/конфигов.

## Пошаговая реализация

1. **Сверка требований из документации и планов**

- Пройти документы: [docs/README.md](docs/README.md), [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md), [docs/DEPLOYMENT.md](docs/DEPLOYMENT.md), [docs/DATA_DICTIONARY.md](docs/DATA_DICTIONARY.md), [docs/RUNBOOK.md](docs/RUNBOOK.md), [docs/SECURITY.md](docs/SECURITY.md), [docs/ADMIN_GUIDE.md](docs/ADMIN_GUIDE.md), [docs/ADR/000-template.md](docs/ADR/000-template.md).
- Свести требования в проверочный список «требование -> подтверждение в коде/конфиге -> статус».
- Сверить с планами: [.cursor/plans/delivery_assistant_mvp_1a5e90e4.plan.md](.cursor/plans/delivery_assistant_mvp_1a5e90e4.plan.md), [.cursor/plans/delivery_assistant_mvp_d94a7f74.plan.md](.cursor/plans/delivery_assistant_mvp_d94a7f74.plan.md), [.cursor/plans/delivery_assistant_mvp_e05fc5ee.plan.md](.cursor/plans/delivery_assistant_mvp_e05fc5ee.plan.md).

1. **Максимально глубокий аудит кода (по всем модулям)**

- Проверить слои и контракты: [src/bot](src/bot), [src/core](src/core), [src/infra](src/infra), [migrations](migrations), [tests](tests).
- Зафиксировать по каждому файлу: назначение, реализованные сценарии, недостающие ветки/валидации/ошибки/покрытие тестами.
- Отдельно проверить критические места:
  - [src/infra/queue/tasks.py](src/infra/queue/tasks.py) (реальность ingest-пайплайна),
  - [src/bot/admin/menu.py](src/bot/admin/menu.py) (заглушки и RBAC),
  - [src/infra/integrations](src/infra/integrations) (пустые интеграции),
  - [pyproject.toml](pyproject.toml) (корректность метаданных `readme`).

1. **Диагностика и починка MCP Context7**

- Проверить и поправить конфиг MCP: [.cursor/mcp.json](.cursor/mcp.json).
- Синхронизировать правила/ожидания: [.cursorrules](.cursorrules), [.cursor/rules/70-System prompt.md](.cursor/rules/70-System prompt.md) (если применимо).
- Убрать секреты из репозиторных шаблонов: [.env.example](.env.example).
- Провести smoke-проверку MCP в IDE: подтверждение, что запросы к документации библиотек реально идут через Context7.

1. **Проверка работоспособности после правок**

- Прогнать инфраструктурные и функциональные проверки:
  - сборка/подъём сервисов,
  - миграции,
  - тесты `pytest`,
  - базовые сценарии бота (`/start`, `/admin`, FSM потоки),
  - проверка обработки retry/429 в очередях.
- Исправить обнаруженные критичные регрессии, если они появляются после изменений.

1. **Итоговый отчёт для тебя (простым языком)**

- По каждому модулю дать статус: `Реализовано` / `Частично` / `Не реализовано`.
- Отдельно выдать блок `Что мешает продакшену` (критичные риски) и `Что делать дальше` (приоритизированный roadmap).
- Подготовить короткий “план следующего шага” без перегруза терминами.

## Ожидаемый результат

- Рабочий и безопасный MCP Context7 в проектной среде.
- Полная и понятная карта текущего состояния проекта по факту кода.
- Чёткий список недостающих задач и приоритетов до production-ready состояния.
